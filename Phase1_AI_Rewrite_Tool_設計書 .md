## 4. AI戦略：二段階構成（Groq → Claude）

### 概要

| フェーズ | 対象期間 | AI基盤 | モデル | コスト |
|----------|----------|--------|--------|--------|
| **Phase A（立ち上げ期）** | リリース〜ユーザー500人 | Groq API 無料枠 | Llama 4 Scout 17B / Llama 3.3 70B | ¥0 |
| **Phase B（成長期）** | ユーザー500人〜 | Claude API | claude-sonnet-4-6 | 従量課金 |

### Phase A：Groq API（無料枠）の活用

**なぜGroqから始めるか**
- クレジットカード不要、即座にAPIキー取得可能
- 推論速度が極めて速い（LPUチップによるハードウェア最適化）
- 初期の開発・テスト・ローンチ時のAPIコストが完全に¥0

**Groq 無料枠のレート制限**

| モデル | RPM | RPD | TPM | TPD |
|--------|-----|-----|-----|-----|
| Llama 4 Scout 17B | 30 | 1,000 | 30,000 | 500,000 |
| Llama 3.3 70B | 30 | 1,000 | 12,000 | 100,000 |

※ RPM=リクエスト/分、RPD=リクエスト/日、TPM=トークン/分、TPD=トークン/日

**推奨モデル使い分け**
- リライト・トーン変換 → Llama 3.3 70B（高品質・日本語対応良好）
- 翻訳 → Llama 4 Scout 17B（高速・トークン上限が大きい）

**無料枠で対応可能なユーザー数の目安**
- 1回のリライトで約500トークン消費と仮定
- Llama 3.3 70B: 1日100,000TPD ÷ 500 = 約200リクエスト/日
- ユーザーが平均3回/日使用 → 約60〜70人のDAUに対応可能
- Freeプラン（1日5回制限）のみであれば、DAU 200人以上も可能

### Phase B：Claude APIへの切替

**切替のタイミング**
- DAUが100人を超え、Groq無料枠のレート制限に頻繁に到達するようになったら
- Pro会員が増え、より高品質な出力が求められるようになったら
- 収益がAPIコストを十分にカバーできるようになったら（月売上¥50,000以上目安）

**Claude APIの利点（Phase B）**
- 日本語のリライト品質がトップクラス
- ニュアンスの理解力が高く、トーン変換の精度が優れている
- 安定した品質で、Proユーザーの満足度を維持できる

### AI Router（プロバイダ切替）の実装

バックエンドにAI Routerレイヤーを設けることで、フロントエンドのコード変更なしにAIプロバイダを切り替えられるようにします。

```typescript
// packages/shared/src/ai-router.ts

type AIProvider = "groq" | "claude";

interface RewriteRequest {
  text: string;
  mode: "rewrite" | "tone" | "translate";
  option: string;
}

// 環境変数でプロバイダを切替
const provider: AIProvider = process.env.AI_PROVIDER as AIProvider || "groq";

async function callAI(request: RewriteRequest): Promise<string> {
  const prompt = buildPrompt(request);

  if (provider === "groq") {
    return callGroq(prompt);    // Groq SDK使用
  } else {
    return callClaude(prompt);  // Anthropic SDK使用
  }
}
```

**切替方法**: 環境変数 `AI_PROVIDER=groq` → `AI_PROVIDER=claude` に変更するだけ。
Vercelのダッシュボードから即座に切替可能。

### プロンプト設計（共通）

Groq（Llama系）でもClaude APIでも同じプロンプトが使えるように設計します。

#### リライトモード

```
あなたは文章リライトの専門家です。
以下のテキストを「{mode}」のスタイルにリライトしてください。

【モード】
- casual: 友人に話すようなカジュアルな文体
- formal: ビジネス文書にふさわしいフォーマルな文体
- polite: 丁寧で敬語を使った文体
- friendly: 親しみやすく温かみのある文体

【ルール】
- 元の意味を変えないこと
- 自然な日本語であること
- リライト結果のみを返すこと（説明不要）

【入力テキスト】
{user_text}
```

#### 翻訳モード

```
あなたはプロの翻訳者です。
以下のテキストを{target_language}に翻訳してください。

【ルール】
- 自然な{target_language}であること
- 直訳ではなく、意訳を心がけること
- 翻訳結果のみを返すこと（説明不要）

【入力テキスト】
{user_text}
```


これにより、APIコストはPro会員の売上で十分カバーできる構造になります。

## 10. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| Groq無料枠のレート制限到達 | ユーザーが使えない時間帯が発生 | リクエストキューイング、ユーザーへの待機表示、Phase Bへの早期移行判断 |
| Groq無料枠の廃止・縮小 | コスト急増 | AI Routerにより即座にClaude APIへ切替可能。常にPlan Bを準備 |
| Claude API障害 | サービス停止 | Groqへのフォールバック（AI Routerで自動切替） |
| APIコスト増大 | 赤字 | Freeユーザー→Groq、Proユーザー→Claudeのハイブリッド戦略 |
| App Store審査リジェクト | リリース遅延 | ガイドライン事前確認、PWAでのWeb先行リリース |
| 競合サービスの出現 | ユーザー流出 | 独自機能（カスタムプロンプト等）で差別化 |
| 個人情報漏洩 | 信頼失墜 | Supabase RLS活用、入力データの非保存オプション |
